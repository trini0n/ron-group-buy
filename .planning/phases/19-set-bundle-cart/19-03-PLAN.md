# Plan 19-03 — Checkout + Order History

## Goal

Process bundle items through checkout into `order_bundle_items`, display bundles in order history, and handle the existing pending order conflict flow for bundles.

## Context

- Order creation uses a Postgres RPC `create_order_with_items` — bundles will be handled separately via a direct INSERT after the RPC succeeds
- `order_bundle_items` columns: `order_id`, `set_code`, `set_name` (snapshot), `quantity`, `price_at_purchase` (snapshot of live `sets.price` at checkout time)
- D3: No stock check for bundles
- Existing pending order flow: "replace" deletes old order and creates new; "merge" adds items to existing order

### Task 3 — Checkout Submission: Include Bundles

**File:** `src/routes/api/orders/+server.ts`

Extend `CreateOrderSchema` to accept optional `bundles` array:
```ts
const BundleItemSchema = z.object({
  setCode: z.string().min(1),
  quantity: z.number().int().min(1)
})

const CreateOrderSchema = z.object({
  // ... existing fields ...
  bundles: z.array(BundleItemSchema).optional()
})
```

After the order is created via `create_order_with_items` RPC and we have `order_id`:

```ts
if (bundles && bundles.length > 0) {
  // Fetch live prices server-side (never trust client)
  const setCodes = bundles.map(b => b.setCode)
  const { data: setRows } = await locals.supabase
    .from('sets')
    .select('set_code, set_name, price')
    .in('set_code', setCodes)

  const setMap = new Map(setRows?.map(s => [s.set_code, s]) ?? [])

  const bundleInserts = bundles
    .map(b => {
      const set = setMap.get(b.setCode)
      if (!set || set.price == null) return null  // skip if set not found or no price
      return {
        order_id: orderId,
        set_code: b.setCode,
        set_name: set.set_name,           // snapshot
        quantity: b.quantity,
        price_at_purchase: set.price      // snapshot of live price at checkout
      }
    })
    .filter(Boolean)

  if (bundleInserts.length > 0) {
    await locals.supabase.from('order_bundle_items').insert(bundleInserts)
  }
}
```

Apply same logic to `mergeIntoExistingOrder` — append new bundles to existing `order_bundle_items` (or increment quantity if set_code already present).

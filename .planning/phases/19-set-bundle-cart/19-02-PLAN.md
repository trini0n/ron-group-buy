# Plan 19-02 — Cart UI + "Add to Cart" on /sets Listing

## Goal

Display bundles in the cart as a distinct section above individual cards, and add an "Add to Cart" button to each set on the `/sets` listing page.

## Context (from 19-CONTEXT.md)

- The existing `/cart` page (or drawer) renders `cartStore.items`; extend it to also render `cartStore.bundles`
- "Add to Cart" button goes on `/sets` listing page — one per set card

## Prerequisites

### Task 2 — Cart Page: Bundles Section

**File:** `src/routes/cart/+page.svelte`

Add a bundles section **above** the existing items list:

```
┌─────────────────────────────────────┐
│  SETS (2)                           │   ← section heading, shown only when bundles.length > 0
│  ┌───────────────────────────────┐  │
│  │ NP Foil 42 Set (65$)    $65   │  │   ← set name, live price
│  │ [setCode badge]  qty: [−][1][+]│  │   ← code, quantity stepper, remove button
│  └───────────────────────────────┘  │
│                                     │
│  INDIVIDUAL CARDS (12)              │   ← existing section, always shown
│  [existing card rows...]            │
└─────────────────────────────────────┘
```

**Bundle row elements:**
- Set name (links to `/sets/[setCode]`)
- `set_code` badge (monospace)
- Live price (`$65.00`) — pulled from `bundle.set.price`
- Quantity stepper: `[−]` `[qty]` `[+]` — calls `cartStore.updateBundleQuantity()`
- Remove button (×) — calls `cartStore.removeBundle()`
- Subtotal for row: `price × quantity`

**Section heading logic:**
```svelte
{#if cartStore.bundles.length > 0}
  <h2>Sets ({cartStore.bundles.length})</h2>
  <!-- bundle rows -->
{/if}

{#if cartStore.items.length > 0}
  <h2 class={cartStore.bundles.length > 0 ? 'mt-6' : ''}>
    Individual Cards ({cartStore.items.length})
  </h2>
  <!-- existing card rows -->
{/if}
```

**Cart total:** `cartStore.total` already includes bundles (from 19-01 store update).

**Empty state:** Only show "Your cart is empty" when BOTH `items` and `bundles` are empty.

### Task 3 — "Add to Cart" Button on /sets listing

**File:** `src/routes/sets/+page.svelte`

Add an "Add Set to Cart" button to each set card. Place it at the bottom of the card, below the price/count row.

```svelte
<button
  onclick={() => addSetToCart(set.set_code, set.set_name)}
  disabled={addingSetCode === set.set_code}
  class="w-full mt-3 ..."
>
  {addingSetCode === set.set_code ? 'Adding…' : 'Add to Cart'}
</button>
```

**`addSetToCart` function:**
```ts
async function addSetToCart(setCode: string, setName: string) {
  addingSetCode = setCode
  try {
    await cartStore.addBundle(setCode, 1)
    toast.success(`${setName} added to cart`)
  } catch {
    toast.error('Failed to add set to cart')
  } finally {
    addingSetCode = null
  }
}
```

**State:**
```ts
let addingSetCode = $state<string | null>(null)
```

**Auth guard:** If not logged in, button should still work — bundles support guest carts (same as card items). No special auth check needed on the button itself.

# Plan 18-01 ‚Äî Public Sets Listing + Detail Pages + Nav Link

## Goal

Ship two public browse pages for vendor-curated sets, plus a nav link. No cart integration (Phase 19).

## Context

- `sets` table: `set_code` (PK), `set_name`, `sort_order`, `price`, `created_at`
- `set_cards` join table: `set_code FK ‚Üí sets`, `card_id FK ‚Üí cards`

### 3. `/sets/[setCode]` Detail Page

**Files:**
- `src/routes/sets/[setCode]/+page.server.ts` ‚Äî load set + associated cards
- `src/routes/sets/[setCode]/+page.svelte` ‚Äî card grid UI

**Server load:**
```ts
// Step 1: fetch set (404 if not found)
supabase.from('sets').select('set_code, set_name, price').eq('set_code', params.setCode).single()

// Step 2: fetch set_cards joined to full card data
supabase.from('set_cards')
  .select('card_id, cards(*)')
  .eq('set_code', params.setCode)
```

**Card sort order (DETAIL-03):**

- Secondary: `cards.collector_number` ASC (numeric-aware: `1 < 2 < 10 < 100`, not string sort)

**UI design:**
- Breadcrumb: `‚Üê Back to Sets` ‚Üí `/sets`
- Header: set name, `set_code` badge, price (`$65.00`), card count subtitle
- `CardGrid` component reused ‚Äî pass the set's cards as the card list
- Each card links to its existing detail page
- Empty state: "No cards in this set yet"
- SEO: `<title>{set_name} ‚Äî Sets ‚Äî Group Buy</title>`

## Sorting Implementation Note

`collector_number` is stored as TEXT. Numeric-aware sort must be done in JS (not DB), since Postgres text sort gives `1, 10, 100, 2`. Pattern:

```ts
function collectorNumberSort(a: string, b: string): number {
  const na = parseInt(a, 10)
  const nb = parseInt(b, 10)
  if (!isNaN(na) && !isNaN(nb)) return na - nb
  return a.localeCompare(b)
}
```

Release dates sorted by mapping `set_code ‚Üí releaseDate` from the existing setReleaseDates endpoint.

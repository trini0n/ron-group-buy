# Plan 20-01: StacksView Component + View Toggle Integration

## Goal

Add a third "Stacks" view mode to `/sets/[setCode]` that groups cards by MTG expansion into
collapsible vertical columns, with inline image expand on row click, Scryfall set symbol icons in
column headers, and a mobile accordion layout. No DB changes. No changes to CardGrid or CardItem.

### Scryfall set icon URL pattern

```
https://svgs.scryfall.io/card-symbols/{set_code_lowercase}.svg
```

Example: `https://svgs.scryfall.io/card-symbols/mh3.svg`
If the SVG 404s (e.g. custom proxies), the `<img>` onerror handler hides the icon element.

### Natural sort (reuse from sets page)

```ts
function naturalSort(a: string, b: string): number {
  const re = /(\d+)|(\D+)/g
  const tokensA = a.match(re) ?? []
  const tokensB = b.match(re) ?? []
  ...
}
```

### Deduplication key

Same card = `card_name + '|' + card_type`. Cards with same name but different finishes are **separate** rows.

### Task 1 — Create `StacksView.svelte`

**File:** `src/lib/components/cards/StacksView.svelte`

#### 1a. Data pipeline (pure derivation, no reactivity tricks needed)

```
cards: Card[]
  → group by (set_code ?? 'unknown') + set_name
  → sort groups: naturalSort(set_name)
  → within each group:
      → deduplicate: Map<card_name+'|'+card_type, { card: Card, count: number }>
          (keep first occurrence as the representative card)
      → sort deduped rows: collectorNumberSort(collector_number)
  → result: Column[]
```

```ts
interface Column {
  setCode: string      // lowercase, for Scryfall icon URL
  setName: string      // display name
  totalCount: number   // sum of all counts (including dupes)
  rows: Row[]
}

interface Row {
  key: string          // card_name + '|' + card_type
  card: Card           // representative card (first occurrence)
  count: number        // how many copies
}
```

#### 1b. Per-column state

Each column tracks independently:
- `expandedKey: string | null` — which row is expanded (null = none)
- `collapsed: boolean` — accordion collapsed state (mobile only; always false on desktop)

Use a `Map<string, { expandedKey: string | null, collapsed: boolean }>` keyed by `setCode`.
Initialise all columns as `{ expandedKey: null, collapsed: false }`.

#### 1e. Image URL resolution per row

Each expanded row resolves its image at render time (not pre-computed for all rows — lazy).
Use a small helper inside the component:

```ts
function resolveImage(card: Card, failedIds: Set<string>): string {
  const ron = getRonImageUrl(card.ron_image_url)
  if (ron && !failedIds.has(card.serial)) return ron
  if (card.scryfall_id) return getScryfallImageUrl(card.scryfall_id, 'normal')
  return '/images/card-placeholder.png'
}
```

Track `failedIds = $state(new Set<string>())`. On `onerror`, add `card.serial` to the set and
force re-render by reassigning: `failedIds = new Set(failedIds)`.

#### 1h. Collector number sort (reuse pattern from set detail server)

```ts
function collectorNumberSort(a: string | null, b: string | null): number {
  const na = parseInt(a ?? '', 10)
  const nb = parseInt(b ?? '', 10)
  if (!isNaN(na) && !isNaN(nb)) return na - nb
  return (a ?? '').localeCompare(b ?? '')
}
```

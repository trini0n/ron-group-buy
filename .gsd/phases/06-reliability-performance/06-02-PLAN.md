<objective>
Add Zod schema validation to every POST handler that reads a request body, so malformed payloads are rejected uniformly at the boundary with structured field errors.

**Pattern used in every handler** (replace manual validation for covered fields):

```ts
import { z } from 'zod'

const schema = z.object({
  /* ... */
})

export const POST: RequestHandler = async ({ request, locals }) => {
  const result = schema.safeParse(await request.json())
  if (!result.success) {
    return json({ error: 'Invalid request body', issues: result.error.issues }, { status: 400 })
  }
  const { field1, field2 } = result.data
  // ... rest of handler unchanged
}
```

Handlers with no request body (`POST /api/orders/[id]/load-to-cart`, `POST /api/admin/inventory/sync`, `POST /api/admin/users/[id]/reset-password`) are unchanged.

Purpose: Closes the input validation boundary for all POST endpoints — rejects type coercion, extra fields, and missing required fields before any DB write.

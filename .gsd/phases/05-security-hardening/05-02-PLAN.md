After the existing phone number null/empty check (around line 54–57), add E.164 format validation:

```ts
// Existing check (keep as-is):
if (!phoneNumber || !String(phoneNumber).trim()) {
  throw error(400, 'Phone number is required')
}

// Add immediately after:
const e164Regex = /^\+[1-9]\d{1,14}$/
if (!e164Regex.test(String(phoneNumber).trim())) {
  throw error(400, 'Invalid phone number format. Must be E.164 (e.g. +15551234567)')
}
```

E.164 rules enforced:

- Must start with `+`
- First digit after `+` must be 1–9 (no leading zero)
- Total length 2–15 digits after `+` (E.164 max is 15 digits total including country code)
- Full regex: `/^\+[1-9]\d{1,14}$/`

Do NOT modify any other part of the orders handler.

<task type="auto">
  <name>Task 2: Add SSRF final hostname check in gphoto-converter.ts</name>
  <files>src/lib/server/gphoto-converter.ts</files>
  <action>
Inside `getDirectPhotoUrl()`, after the `if (!response.ok)` block and before `const html = await response.text()`, add a hostname validation check:

```ts
// After: if (!response.ok) { ... }

// SSRF guard: validate the final resolved URL hostname after redirects
const ALLOWED_GPHOTO_HOSTNAMES = new Set(['photos.google.com', 'lh3.googleusercontent.com'])
const finalHostname = new URL(response.url).hostname
if (!ALLOWED_GPHOTO_HOSTNAMES.has(finalHostname)) {
  logger.error(
    { shareUrl, finalUrl: response.url, finalHostname },
    'SSRF guard: unexpected hostname after redirect — aborting'
  )
  return null
}

// Existing: const html = await response.text()
```

Place `ALLOWED_GPHOTO_HOSTNAMES` as a module-level constant (above the `getDirectPhotoUrl` function) rather than inside the function body, to avoid recreating it on every call:

```ts
const GOOGLE_CONTENT_PREFIX = 'https://lh3.googleusercontent.com/'
const ALLOWED_GPHOTO_HOSTNAMES = new Set(['photos.google.com', 'lh3.googleusercontent.com'])
```

The check uses `response.url` which is the final URL after all HTTP redirects are followed by `fetch`. If a `photos.app.goo.gl` short-link redirects to an unexpected host (SSRF attempt), this returns null and logs the incident instead of fetching and parsing attacker-controlled HTML.
</action>

<done>SSRF hostname guard added; getDirectPhotoUrl() returns null and logs if final response URL is not photos.google.com or lh3.googleusercontent.com.</done>

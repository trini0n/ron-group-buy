# Plan 04-02: Svelte 5 Reactivity + Notification Fix

## Objective

Resolve all `state_referenced_locally` warnings in Svelte 5 runes mode and fix the `is_active: boolean | null` compile error in the notification settings dialog.

## Context

After DB type regen (04-01), two categories of issues remain. First, `$state({...})` initializers that reference other reactive state values cause Svelte 5 `state_referenced_locally` warnings — these are resolved by wrapping the initializer in `untrack()`. Second, the `NotificationTemplate` interface declares `is_active: boolean` but the DB column is nullable, so any edit dialog binding to it produces a compile error; the fix is widening the type and adding `?? false` guards at boolean use sites.

Fix `src/routes/+page.svelte` — wrap the `prevFilters` `$state({...})` initializer in `untrack(() => (...))` to suppress the 8 `state_referenced_locally` warnings (lines ~57–65). `untrack` is already imported in this file.
```svelte
// BEFORE
let prevFilters = $state({
  setCodes: filters.setCodes.join(','),
  colorIdentity: filters.colorIdentity.join(','),
  colorIdentityStrict: filters.colorIdentityStrict,
  priceCategories: filters.priceCategories.join(','),

  frameTypes: filters.frameTypes.join(','),
  inStockOnly: filters.inStockOnly,
  isNew: filters.isNew,

// AFTER
let prevFilters = $state(untrack(() => ({
setCodes: filters.setCodes.join(','),
colorIdentity: filters.colorIdentity.join(','),
colorIdentityStrict: filters.colorIdentityStrict,
priceCategories: filters.priceCategories.join(','),
cardTypes: filters.cardTypes.join(','),

inStockOnly: filters.inStockOnly,

viewMode: viewMode
})));

````

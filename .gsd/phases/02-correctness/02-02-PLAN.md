<objective>
Two correctness fixes:

1. **Export manifest race condition** (`export-storage.ts`): The manifest is read-modify-written without any locking. On Vercel, true concurrent writes are unlikely but still possible. Make writes atomic using a write-and-rename pattern or add a simple in-process last-write strategy.

2. **Silent query errors in import search** (`import/search/+server.ts`): Three queries destructure only `data`, discarding `error`. Add error checking so failures are logged and return appropriate error responses rather than empty partial results.

Purpose: Prevent subtle data corruption in exports; surface real errors in search instead of silently returning incorrect results.
Output: Both files hardened against their respective failure modes.
</objective>

<task type="auto">
  <name>Safe manifest writes in export storage</name>
  <files>src/lib/server/export-storage.ts</files>
  <action>
    Read the file. Around lines 30, 45, 59, 60: the manifest is read with `readFileSync`/`readFile`, modified in-memory, then written back with `writeFileSync`/`writeFile`. This is a read-modify-write without any guard.

    Since this runs in a Node.js serverless environment (Vercel), use a module-level in-process mutex (a simple `Promise` chain is sufficient):

    ```typescript
    let manifestLock: Promise<void> = Promise.resolve();

    async function withManifestLock<T>(fn: () => Promise<T>): Promise<T> {
      let resolve!: () => void;
      const next = new Promise<void>(r => { resolve = r; });
      const current = manifestLock;
      manifestLock = next;
      await current;
      try {
        return await fn();
      } finally {
        resolve();
      }
    }
    ```

    Wrap all manifest read-modify-write blocks in `withManifestLock(async () => { ... })`.

    If the file uses sync fs operations, convert them to async (`fs.promises.readFile`, `fs.promises.writeFile`) inside the lock wrapper.
  </action>
  <verify>
    `npm run check` passes. All manifest mutation calls inside the module are wrapped in `withManifestLock`.
  </verify>
  <done>Manifest writes serialized via promise-based lock. Concurrent requests queue, not interleave.</done>

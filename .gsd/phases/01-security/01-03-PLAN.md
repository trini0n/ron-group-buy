<objective>
Two simple security fixes in one plan:

1. **Cleanup endpoint** (`exports/cleanup/+server.ts`): Change fail-open auth (runs unauthenticated when CRON_SECRET missing) to fail-closed (returns 503 when secret not configured, 401 on mismatch).

2. **Import search endpoint** (`import/search/+server.ts`): Replace service-role `createAdminClient()` with `locals.supabase` — RLS allows public reads on `cards` table so service-role is unnecessary and increases attack surface.

Purpose: Eliminate two low-effort high-impact security issues — one that allows unauthenticated admin operations in misconfigured environments, one that unnecessarily bypasses RLS on a public route.
Output: Both files patched. No functional change for correctly-configured environments.
</objective>


    Replace this with a fail-closed pattern where BOTH missing secret AND wrong token are hard failures:

    ```typescript
    const cronSecret = process.env.CRON_SECRET?.trim();
    if (!cronSecret) {
      throw error(503, 'CRON_SECRET not configured');
    }
    const authHeader = request.headers.get('authorization');
    if (authHeader !== `Bearer ${cronSecret}`) {
      throw error(401, 'Unauthorized');
    }
    // cleanup runs only here — after both checks pass
    ```

    The `request` object is available in the RequestHandler parameters — make sure it's destructured from the event parameter if not already. Import `error` from `@sveltejs/kit` if not already imported.

<objective>
Fix open redirect vulnerability in auth callback by sanitizing the `next` query parameter before using it in any redirect. Apply secondary hardening to the conflict page which can also receive a `returnTo` param directly.

Purpose: Prevent phishing via crafted auth callback URLs (e.g. `/auth/callback?next=https://evil.com`)
Output: Both files validate redirect targets are same-origin relative paths before use
</objective>
